## Team 13
# Prioritize - Smart Scheduler

**Prioritize** is an intelligent task management and scheduling application designed to help students optimize their study time. Unlike standard to-do lists, Prioritize uses a weighted algorithm to automatically generate study blocks based on deadlines, difficulty, and grade weight. It also features social collaboration tools to find mutual free time with study partners.

## Key Features

### 1. User Task Management
Users can manage their academic workload with detailed metadata.
*   **CRUD:** Create, Read, Update, and Delete tasks.
*   **Task Details:** Tasks track Name, Deadline, Grade Weight (%), Difficulty (Easy/Medium/Hard), and Estimated Hours.
*   **Auto-Update:** Changes to tasks trigger an automatic regeneration of the schedule within seconds.

### 2. Smart Scheduling Algorithm
The core of the application automates the planning process.
*   **Logic:** The system calculates a `Priority Score` using the formula:
    > `(Weight × Difficulty Multiplier) / Hours Until Deadline`
*   **Allocation:** High-priority tasks are automatically slotted into the user's defined "Availability" blocks.
*   **Reasoning:** Each scheduled block displays a priority score and other information (e.g., *"Priority Score: 2.31, Deadline: 2025-12-02 23:59, Weight: 28%, Allocated: 3.76 hours"*).

### 3. Study Timer + Analytics
A built-in productivity tool to track actual work versus planned work.
*   **Real-Time Timer:** Users can Start and Stop a timer for specific tasks.
*   **Auto-Logging:** Stopping the timer automatically saves the session data to the database (no manual entry required).
*   **Dashboard:** Visual analytics showing:
    *   Total hours logged per task (e.g., *"CMPT276: 8.5 hours"*).
    *   Breakdown by day (Today, Last 7 Days, Last 30 Days).

### 4. Calendar Integration
Export capabilities for external notifications.
*   **iCal Export:** Users can download a `.ics` file containing their generated schedule.
*   **Compatibility:** The file imports successfully into Google Calendar, Apple Calendar, and Outlook, preserving task details and priority score in the event description.

### 5. Collaboration Mode
Social features for coordinating study sessions.
*   **Friend System:** Users can send, accept, and decline friend requests using unique User IDs.
*   **Schedule Viewing:** Friends can view each other's generated schedules.
*   **Mutual Free Time:** A backend algorithm that identifies overlapping free time by:
    1.  Fetching recurring availability for both users.
    2.  Subtracting scheduled busy blocks.
    3.  Calculating the mathematical intersection to display results like *"Free Sunday 14:35 - 16:34"*.

## Event_service

The system tracks **User Engagement** to measure platform usage.

*   **Metric:** `Total Session Duration` (User Online Time).
*   **Methodology:**
    1.  Records `session_start` timestamp on Login.
    2.  Records `session_end` timestamp on Logout or Session Expiry.
    3.  Calculates `total_online_time` and stores it in the analytics database.
*   **Purpose:** Distinguishes between "Time spent browsing/planning" vs. "Time spent actively studying" to better understand user behavior.

## How to Run Locally

1.  **Create Environment Variables**
    Create a file named `.env` in the root directory. Copy these default values (pre-configured for local Docker):
    ```env
    POSTGRES_USER=postgres
    POSTGRES_PASSWORD=password
    POSTGRES_DB=team13
    DATABASE_HOST=db
    REDIS_URL=redis://redis:6379
    AUTH_TTL_SECONDS=3600
    # JWT Keys will be auto-generated by the app if left blank locally.
    ```

2.  **Access the Application**
    *   **Frontend Dashboard:** [http://localhost:8000/index.html](http://localhost:8000/index.html)
    *   **Swagger API Docs:** [http://localhost:8000/docs](http://localhost:8000/docs)
    *   **Admin Panel:** [http://localhost:8000/admin/](http://localhost:8000/admin/)
        *   *Admin Password:* `team13`
